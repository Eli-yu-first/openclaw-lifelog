# OpenClaw LifeLog 架构设计

## 系统概述

OpenClaw LifeLog 是一个基于 OpenClaw 框架的个人生活记录与智能助理系统，通过多模态感知、行为分析和多智能体协作，实现对用户日常习惯的全面记录和主动式智能提醒。

## 核心模块

### 1. 数据采集模块

**职责**: 通过摄像头捕获用户行为视频

**技术实现**:
- 使用 OpenClaw 的 `camera.clip` 命令
- 通过 Cron Job 定时触发采集
- 支持 macOS/iOS/Android 平台

**数据流**:
```
摄像头 → Base64 编码 → HAR 服务
```

### 2. 行为分析模块 (HAR Service)

**职责**: 将视频数据转化为结构化行为数据

**技术实现**:
- 基于 Flask 的 RESTful API
- 支持 YOLOv8-Pose 姿态估计
- Docker 容器化部署

**API 接口**:
```
POST /analyze
{
  "video": "<base64>",
  "timestamp": "2026-02-03T08:15:30Z"
}

Response:
{
  "activity": "brushing_teeth",
  "duration_seconds": 120,
  "confidence": 0.92
}
```

### 3. 多智能体协作层

**职责**: 分层管理用户习惯数据

**Agent 层级**:
```
MasterHabitAgent
├── MorningRoutineAgent
├── DentalHygieneAgent
├── WorkLifeAgent
├── DepartureAgent
└── HealthAgent
```

**通信机制**:
- 使用 `sessions_spawn` 分发任务
- 子 Agent 独立运行，结果公告回主 Agent

### 4. 长期记忆模块

**职责**: 存储和查询用户习惯数据

**存储结构**:
```
~/.openclaw/workspace-{agent}/
├── MEMORY.md           # 习惯总结
└── memory/
    └── YYYY-MM-DD.md   # 每日记录
```

**查询方式**:
- `memory_search` 语义搜索
- 支持混合检索（向量 + 文本）

### 5. 主动提醒模块

**职责**: 基于习惯数据发送智能提醒

**实现方式**:
- OpenClaw Cron Jobs
- 定时检查 + 条件判断 + 消息发送

**提醒场景**:
| 场景 | 触发时间 | 检查内容 |
|------|---------|---------|
| 忘记刷牙 | 9:00 AM | 早上刷牙记录 |
| 久坐提醒 | 每 2 小时 | 活动记录 |
| 出门检查 | 8:00 AM | 物品携带 |

## 数据流

```
1. 采集: 摄像头 → DataAcquisitionAgent → 视频数据
2. 分析: 视频数据 → HAR Service → 结构化行为
3. 分发: MasterHabitAgent → sessions_spawn → 子 Agent
4. 存储: 子 Agent → memory/YYYY-MM-DD.md
5. 查询: Cron Job → MasterHabitAgent → memory_search
6. 提醒: MasterHabitAgent → 消息通道 → 用户
```

## 安全考虑

1. **数据隐私**: 所有数据本地存储，不上传云端
2. **权限控制**: 摄像头访问需用户明确授权
3. **Agent 隔离**: 子 Agent 在独立沙箱中运行

## 扩展性

系统设计支持以下扩展:

1. **新增 Agent**: 添加新的生活场景管理
2. **新增行为**: 扩展 HAR 服务支持的行为类型
3. **新增提醒**: 配置新的 Cron Job
4. **新增通道**: 接入更多消息平台
